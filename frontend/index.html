公开页 Worker 部署 把扫码公开页放到 Cloudflare 边缘

扫码公开页（/p/<token>）默认由当前 FastAPI 后台直接返回。把它部署到 Cloudflare Workers 可以获得：

  • 全球边缘缓存，扫码秒开（<50ms）
  • 30 天 TTL 缓存，远超 7 天邮寄窗口
  • 公开流量不打到你的服务器
  • 重新生成 token 后旧 URL 立即失效（版本化缓存）

① Worker 域名

填写部署好的 Worker 公网地址（含 https://，不带尾部斜杠）。留空则 QR 用当前后台域名。保存后立即生效。

示例：https://card.your-domain.com。留空表示不部署 Worker，QR 用当前后台域名。

② 部署方式（推荐 GitHub 自动部署）

本仓库的 worker-src/ 目录已经包含部署所需的所有文件（Worker 代码 + wrangler.toml + package.json + README）。

方式 A：GitHub 自动部署（推荐）

  1. Cloudflare 控制台 → Workers & Pages → Create → Pages → Connect to Git
  2. 选择仓库 dttxorg/giffgaff-label-manager
  3. 配置：
    • Production branch: main
    • Build command: 留空
    • Build output directory: 留空
    • Root directory (advanced): worker-src
  4. 点 Save and Deploy（首次会失败因为没设环境变量）
  5. 进项目 Settings → Environment variables → 加：
    • Key: API_BASE · Value: https://label.example.com（你 admin 的实际 URL，不要带尾部斜杠）
  6. 回到 Deployments → Retry deployment
  7. 部署成功后，去 Settings → Triggers → Custom Domains → Add Custom Domain → 填 card.example.com（或你想用的子域）
  8. 回到本页面顶部「① Worker 域名」填 https://card.example.com，点保存
  9. 以后改代码 git push 到 main，Cloudflare 自动部署新版本

方式 B：手动 wrangler CLI

如果你想完全手动控制部署：

    1. 本机安装 wrangler（Cloudflare 官方 CLI）： 
      npm install -g wrangler
    2. 登录 Cloudflare 账号（会跳浏览器）： 
      wrangler login
    3. 创建 Worker 项目： 
      mkdir public-card && cd public-card
wrangler init --type javascript --no-git
    4. 把下面的 Worker 代码 粘贴到 src/index.js（覆盖默认内容）：
      src/index.js
    5. 把下面的 wrangler.toml 粘贴到项目根目录（API_BASE 已自动填入当前后台域名）：
      wrangler.toml
    6. 部署到 Cloudflare： 
      wrangler deploy

      首次部署会要求选子域（workers.dev），按提示选一个即可。

    7. 绑定你的自定义域名（让客户扫码用你自己的域名）：
      1. 打开 Cloudflare 控制台 → Workers & Pages
      2. 点 public-card → 上方 Settings → Triggers
      3. Custom Domains → Add Custom Domain
      4. 填入 card.example.com（你的子域），点 Add
      5. Cloudflare 自动加 DNS（CNAME），等 30 秒生效
    8. 回到本页面顶部 ① Worker 域名，填入 https://card.example.com，点保存。

    ③ 验证

    任意客户的「已激活号码资料扫码页」卡片里点"在新标签打开"，应该：

    • 页面正常显示 giffgaff 手机号码、初始注册邮箱和复制按钮（card.example.com 域名）
    • F12 → Network → 该请求响应头里有 X-Cache: HIT（第二次刷新后）
    • 点"重新生成公开链接"后，旧 URL 立即返回 404

    ④ 常见问题

    wrangler deploy 报"Authentication error [code: 10000]"

    重新登录：wrangler logout && wrangler login

    "Could not find a Workers script"

    确认 wrangler.toml 里 main = "src/index.js"，且 src/index.js 存在并有 export default。

    扫码后页面 404 / 打不开

    ① 确认「Worker 域名」配置已保存
    ② 浏览器直接访问 https://card.example.com/p/<token> 看是否 404
    ③ 如果 404，检查 Worker 日志：wrangler tail

    缓存多久生效？

    Cloudflare Cache API 硬上限 30 天（免费版也一样）。本方案足够了 —— 7 天邮寄 + 几周扫码窗口远小于 30 天。如果客户一个月后还没扫码，让他联系你重新发卡。
